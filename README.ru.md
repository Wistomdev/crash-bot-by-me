Мощный Discord бот с несколькими режимами работы для управления серверами, создания бэкапов и автоматизации. Имеет графический интерфейс и поддержку командной строки.

**Русская версия** | [English Version](./README.md)

---

## 📋 Возможности

- **Несколько режимов работы:**
  - `pereezd` - Режим переезда (рассылка в ЛС, пересоздание каналов со спамом)
  - `full` - Полный нюк (бан участников, удаление всего, создание голосовых каналов)
  - `light` - Лёгкий режим (бан ботов, удаление каналов/ролей)
  - `audit` - Режим аудита (только бэкап, без деструктивных действий)
  
- **Продвинутые функции:**
  - 🔄 Автоматический бэкап сервера перед действиями
  - 📨 Массовая рассылка в ЛС всем участникам
  - 🔁 Режим бесконечного спама в созданных каналах
  - 🎨 Изменение названия и иконки сервера
  - 🖥️ Графический интерфейс (Electron)
  - 💾 Система бэкапа и восстановления
  - ⚡ Высокоскоростное параллельное выполнение
  - 🤫 Режим невидимки (скрытность)

## 🚀 Установка

### Требования
- **Node.js** 18.0.0 или выше ([Скачать](https://nodejs.org/))
- **Git** ([Скачать](https://git-scm.com/))
- Токен Discord бота ([Как получить](#как-получить-токен-discord-бота))

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/Wistomdev/crash-bot-by-me.git
cd crash-bot-by-me
```

### Шаг 2: Установка зависимостей
```bash
npm install
```

### Шаг 3: Настройка окружения
Скопируйте `.env.example` в `.env`:
```bash
copy .env.example .env
```

Отредактируйте файл `.env`:
```env
DISCORD_TOKEN_1=ваш_токен_discord_бота
CHANNEL_OR_ROLE_NAME=nuked
BOT_MODE=audit
BLOCKED_GUILD_IDS=
```

### Шаг 4: Сборка проекта
```bash
npm run build
```

## ⚙️ Конфигурация

### Переменные окружения (`.env`)

| Переменная | Описание | По умолчанию | Пример |
|-----------|----------|--------------|--------|
| `DISCORD_TOKEN_1` | Токен Discord бота | **Обязательно** | `MTIzNDU2Nzg5...` |
| `CHANNEL_OR_ROLE_NAME` | Имя для создаваемых каналов/ролей | `nuked` | `hacked` |
| `BOT_MODE` | Режим работы | `audit` | `pereezd`, `full`, `light` |
| `BLOCKED_GUILD_IDS` | ID серверов для пропуска (через запятую) | - | `123456789,987654321` |

### Режимы работы

#### 🔷 `audit` - Режим аудита (Безопасный)
- Создаёт бэкап всех серверов
- Не выполняет деструктивных действий
- Идеально для тестирования и бэкапа

#### 🔷 `pereezd` - Режим переезда
1. Изменяет название и иконку сервера (опционально)
2. Отправляет ЛС всем участникам
3. Удаляет все роли
4. Удаляет все каналы
5. Создаёт 200 текстовых каналов
6. Бесконечный спам во всех каналах

#### 🔷 `full` - Режим полного нюка
1. Банит всех участников (кроме ботов)
2. Удаляет все каналы
3. Удаляет все роли
4. Создаёт 200 голосовых каналов

#### 🔷 `light` - Лёгкий режим
1. Банит всех ботов (кроме белого списка)
2. Удаляет все каналы
3. Удаляет все роли

## 🔑 Как получить токен Discord бота

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications)
2. Нажмите **"New Application"**
3. Введите название приложения и создайте
4. Перейдите в раздел **"Bot"** в левом меню
5. Нажмите **"Reset Token"** и скопируйте токен
6. Включите **Privileged Gateway Intents:**
   - ✅ PRESENCE INTENT
   - ✅ SERVER MEMBERS INTENT
   - ✅ MESSAGE CONTENT INTENT
7. Перейдите в **"OAuth2"** → **"URL Generator"**
8. Выберите scope:
   - ✅ `bot`
   - ✅ `applications.commands`
9. Выберите права бота:
   - ✅ Administrator (или конкретные права)
10. Скопируйте сгенерированную ссылку и откройте в браузере
11. Выберите сервер и авторизуйте бота

## 🐛 Решение проблем

### Бот не подключается
- ✅ Проверьте токен в `.env`
- ✅ Убедитесь что бот приглашён на сервер
- ✅ Проверьте что Privileged Intents включены

### Ошибки "Missing Access"
- ✅ Убедитесь что у бота права Administrator
- ✅ Роль бота должна быть выше целевых ролей
- ✅ Бот не может управлять владельцем сервера

### Ошибки rate limit
- ✅ Уменьшите `concurrency` в коде
- ✅ Увеличьте задержки между действиями
- ✅ Это нормально для больших операций

### GUI не запускается
- ✅ Сначала выполните `npm run build`
- ✅ Проверьте версию Node.js (18+)
- ✅ Переустановите зависимости: `npm install`

## 📝 Расширенная конфигурация

### Кастомные настройки в коде

Отредактируйте `src/index.ts` для настройки поведения:

```typescript
const options: NukeBotOptions = {
  channelName: 'nuked',           // Название канала
  roleName: 'nuked',              // Название роли
  messageText: 'Server nuked!',   // Сообщение для спама
  mode: 'pereezd',                // Режим работы
  leaveImmediately: false,        // Немедленно покинуть серверы
  concurrency: 10,                // Параллельные операции (1-50)
  infiniteSpam: true,             // Включить бесконечный спам
  spamDelayMs: 0,                 // Задержка между сообщениями (0 = нет)
  createBackup: true,             // Создать бэкап перед действиями
  dmMembers: true,                // Отправить ЛС участникам
  dmDelayMs: 1000,                // Задержка между ЛС (мс)
  changeServerName: 'NUKED',      // Новое название сервера (опционально)
  changeServerIcon: 'URL',        // URL новой иконки (опционально)
};
```

### Настройки concurrency
- `1-5` - Безопасно, медленно
- `10-20` - Сбалансировано
- `30-50` - Быстро, могут быть rate limit

### Настройки спама
- `spamDelayMs: 0` - Максимальная скорость (может вызвать rate limit)
- `spamDelayMs: 100` - Быстро с меньшими rate limit
- `spamDelayMs: 1000` - Безопасно, 1 сообщение в секунду
